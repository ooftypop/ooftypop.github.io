What Is A Web Application?

by Kevin J. Storberg

Published on: February 7, 2019

When I tell people I am a Web Application Developer, I often find that my audience does not know what I do for a living. In fact, before I explained the nature of my profession to one of my friends, he believed I “doodled on photoshop all day long”. My friend is not alone. I have encountered a great deal of individuals that do not actually know what a web application developer does, and it is easy to realize that by extension, they do not know what a web application is.

This should not surprise anyone though. It is not necessarily a non-developer’s job to understand what a web application is, or how we developers build them. Indeed, applications like Wix and Wordpress confuse the layman into believing that websites and web apps are interchangeable terms. The job of recognizing the difference belongs to the professional web application developer.

However, I feel it is in everyone’s best interest to be able to tell the difference. How else would someone know when they can use a do-it-yourself application to launch a simple website (like a personal blog), or when recruiting a professional web application development firm is necessary?

As it turns out, there is no ubiquitously accepted definition of web application. Particularly because there exists a gray area between web application and “dynamic-web-page”. But the good news is that identifying a web application is fairly intuitive once you wrap your head around what a web application does.

Let’s take a look at the definition of a web app according to Wikipedia:

"In computing, a web application or web app is a client–server computer program which the client (including the user interface and client-side logic) runs in a web browser. Common web applications include webmail, online retail sales, and online auction."

I think that is a great definition, but it could be a lot to unpack. Especially if you are not computer literate. Let's see if we can simplify this.

"In computing, a web application or web app is a client–server computer program . . ."

For our purposes, we can throw out superfluous language like “In computing”, and “client–server”. Which basically leaves us with: a web application is a computer program.

“which the client (including the user interface and client-side logic) runs in a web browser.”

Again, we can ignore some superfluous language like “the client (including the user interface and client-side logic)”. Leaving us with: which runs in the browser. The browser is what we use to access the internet. If we substitute that idea we have: a web application is a computer program, which we access on the internet.

This definition makes sense of Wikipedia’s example cases: Webmail, online-retail-sales, and online-auctions require sophisticated software to manage the many moving parts of remote communication and/or transactions (e.g. user profiles, addresses, inventory, fund transfers, vendor ratings, conversation history and searching etc). These tasks could never be performed by a simple website like a personal blog.

Applications tend to be built by organizations as custom solutions to unique problems. These problems could be internal; for example the application may improve efficiency or profit margin by decreasing the costs of transactions or tracking client relationships. These problems could also be external, burdening society in some way; in which case the organization may be building the application as a proprietary software which it intends to license out to clients.

Let’s recap. We have a simple definition of web application: a computer program, which we access on the internet. And we know what applications do: Applications tend to be built by organizations as custom solutions to unique problems.

Now when someone asks us what a web application is, we have a way to describe it to them. And by extension, we now know what a web application development firm does, and when it is appropriate to hire one.
