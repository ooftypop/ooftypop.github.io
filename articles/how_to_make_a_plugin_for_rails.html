How To Publish A Rails Plugin, In Four Easy Steps

by Kevin J. Storberg

Published on: January 2, 2019

Rails is a pretty robust framework. I often find myself in awe at just how incredible it really is. But let's face it, sometimes you need something more.

Recently, I was thinking it would be terrific to log instances of my models to the console in a more readable format. I am finicky about code and content being easy to read, thus it was only natural that I extended that disposition. So I wrote a really cool module which I named "SnoopDogg". It alphabetizes the columns in my models and neatly prints them to the console, color coded by data type.

I was quite satisfied with my work and decided that it would be a terrific little snippet to share with the Rails community at large. So I turned it into a plugin which can be found here.

This had me wondering how many cool modules and features have been created by developers and never shared, simply because they did not realize how easy it really is. So I decided to write this post to explain how to generate a simple Rails plugin, hoping that it will lead to some cool new gems for the community to use.

I outline the steps to generate, and publish the gem below (excluding testing). These steps assume you already have the latest version of Rails and Ruby installed.

Step 1: Name your gem

Naming your gem is an important step because the community has naming conventions for ruby gems. Naming conventions give fellow developers a clue about what is under a gem's hood.

In general, we try to follow these pretty simple rules:
  • Use underscore for multiple words
  • Use dashes for extensions
  • Don't use uppercase letters

If you want to read more about naming conventions, you can do so here.

SnoopDogg was pretty simple. Camelcase modules and classes are converted into snake_case when naming a gem. Hence, SnoopDogg => snoop_dogg.

Step 2: Generate The Repository

The step is probably the easiest. Use your shiny new name on Github or some other origin to create a repo. Then, in your local terminal, run "rails plugin new gem_name". In my case, "rails plugin new snoop_dogg".

Poof! You have the scaffolding for a fancy new gem! It still has that new gem smell! Now initialize a local git repo, and push your initial commit to origin like you would do for any other project.

Step 3: Writing The Code

This is where it can get tricky, depending on the complexity of the code you intend to share. We are going to look at a fairly simple case with SnoopDogg, but the general principles can be extended for more complex cases.

In your gem's directory, you will find a file called "gem_name/lib/gem_name.rb". This is where you will place the code you intend to share. For SnoopDogg, this could have been simple enough, all I needed to do was paste my already written module. However, I wanted to go a step further and automatically include the SnoopDogg module in every model of an application downloading the gem. Here is simple way to do just that:

Just under the module, in "snoop_dogg/lib/snoop_dogg.rb", I wrote:

ActiveRecord::Base.send(:include, SnoopDogg)

Boom! Now SnoopDogg will automatically be included in every model in an application which uses it.

Step 4: Publish

Finally! Time to share our gem with the world. In "gem_name/gem_name.gemspec", specify some of the important information about your gem.

Include:
  • The author(s)
  • The homepage
  • A contact email
  • A brief description
  • Any dependencies

Now in "gem_name/lib/gem_name/version.rb" update the version number to reflect your work and then push your shiny new gem to origin.

And that's it! You are done! It is as easy as 1, 2, 3, 4. (That is how that saying goes right?).
